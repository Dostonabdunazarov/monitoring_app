# Система мониторинга IoT-устройств — План

## 1. Обзор

Этот проект представляет собой платформу мониторинга IoT-устройств, предназначенную для сбора, обработки, хранения и визуализации телеметрических данных от распределенных устройств в реальном времени.

---

## 2. Высокоуровневая архитектура

```
[ IoT-устройства ]
        |
   (MQTT / HTTP)
        |
   [ Слой приема данных ]
        |
   [ Брокер сообщений (Kafka) ]
        |
   +------------------------+
   |                        |
[ Потоковая обработка ] [ Сервис записи в хранилище ]
   |                        |
[ Оповещения / Правила ] [ PostgreSQL / TSDB ]
   |                        |
   +-----------+------------+
               |
         [ Backend API ]
               |
         [ Frontend UI ]
```

---

## 3. Компоненты

### 3.1 Слой приема данных
- Принимает данные от IoT-устройств
- Протоколы: HTTP (на старте), MQTT (позже)
- Технология: ASP.NET Core Web API

### 3.2 Брокер сообщений
- Kafka для потоковой передачи событий
- Разделяет слой приема данных и обработку

### 3.3 Потоковая обработка
- .NET Worker Services
- Kafka Consumers
- Зоны ответственности:
  - Фильтрация
  - Агрегация
  - Обнаружение аномалий
  - Срабатывание оповещений

### 3.4 Слой хранения
- PostgreSQL (на старте)
- TimescaleDB (рекомендуется для временных рядов)

### 3.5 Backend API
- ASP.NET Core Web API
- Предоставляет данные для frontend

### 3.6 Frontend
- Alpine.js / React / Vue
- Обновления в реальном времени через SignalR

### 3.7 Система оповещений
- Движок на основе правил
- Примеры условий:
  - Превышен температурный порог
  - Устройство не в сети

## 6. Поток данных

1. Устройство отправляет данные (HTTP/MQTT)
2. API приема данных публикует их в Kafka
3. Kafka распределяет события:
   - Consumer #1 -> сохраняет в БД
   - Consumer #2 -> обрабатывает оповещения
4. Backend API отдает данные
5. SignalR отправляет обновления в UI
